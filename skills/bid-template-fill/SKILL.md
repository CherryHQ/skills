# 标书模板填充 SKILL

## 概述

将脱敏 DOCX 模板填充为可提交的投标文件。覆盖设备采购类投标的商务标 + 技术标 + 报价书全部章节。

## 智能提取流程（新增 v2.3）

技能现在支持直接读取公司信息 DOCX 和招标 PDF/DOCX，自动提取并映射到模板：

`
公司信息.docx ──→ extract_company.py ──→ company.json（自动映射脱敏Key）
招标文件.pdf  ──→ extract_bid.py     ──→ 招标方结构化数据
                     ↓
              bid_pipeline.py ──→ 合并+填充 ──→ 成品标书.docx
`

**一键命令：**
```bash
# 从公司DOCX + 招标PDF 直接生成标书
python bid_pipeline.py 招标文件.pdf --company-docx 公司信息.docx --templates-dir ./模板/
```

> **输出文件夹：**生成的标书文件默认保存在招标文件同目录下的 成品标书/ 子文件夹中，文件命名格式：标书类型_项目名.docx。可通过 --output-dir 自定义。

**分步使用：**
```bash
# 步骤A：提取公司信息
python extract_company.py 公司信息.docx -o company.json

# 步骤B：提取招标信息
python extract_bid.py 招标文件.pdf --print

# 步骤C：管道生成
python bid_pipeline.py 招标文件.md --company-json company.json
```

> **首次使用**：编辑 `desensitize_map.json`，将 `template_desensitize_map` 中的虚拟文本改为你模板中实际出现的脱敏名。也可通过 `--map` 指定自定义映射文件：`python extract_company.py 公司信息.docx --map my_map.json`

## 环境要求

- **Python 方式（推荐）**：Python 3.7+，安装 `python-docx`：`pip install python-docx`
- **Node.js 方式（零依赖备选，仅 Windows）**：Node.js 16+
- **管道方式**：Python 3.7+，`python-docx`，模板需按约定命名

## 工作流

```
步骤1: 读取招标文件 → 提取招标方信息
步骤2: 扫描脱敏模板 → 发现占位符与脱敏文本
步骤3: 收集投标人信息 → 向用户询问必要数据
步骤4: 构建替换数据 → 映射数据到占位符
步骤5: 执行填充 → 调用 fill_py.py 或 fill_template.js
步骤6: 验证输出 → 检查遗漏占位符
```

## 步骤1: 读取招标文件，提取招标方信息

读取招标文件（PDF 或 Markdown/Word），提取并保存为结构化数据：

| 提取项 | 常见模板占位符 | 用途 |
|--------|--------------|------|
| 项目名称 | `[项目名称]`、`_[项目名称]_` | 保证金表、页眉 |
| 招标编号 | `_[招标编号]_`、`[招标编号]` | 投标函、授权书、保证金表 |
| 招标方名称 | `_[招标方名称]_`、`[招标方名称]` | 投标函抬头 |
| 招标代理名称 | `_[招标代理名称]_`、`[招标代理名称]` | 投标函抬头、保证金表抬头 |
| 投标有效期天数 | `【 30 】` | 投标函第4款 |
| 投标保证金金额 | 脱敏金额文本 | 保证金表 |
| 付款方式/质保期/交货期 | — | 差异表 |
| 是否接受联合体/分包 | — | 投标函 |
| 需何种认证 | `_[认证承诺]_` | 投标函第12款 |
| 标段名称 | `（标段1名称）` | 报价总表 |

> 如招标文件为 PDF，使用可用的 PDF 读取工具提取文本。如已有解析好的 Markdown，可直接用 `bid_pipeline.py` 自动提取。

## 步骤2: 扫描脱敏模板，发现占位符

脱敏模板中需要替换的内容分三类：

### 2.1 通用占位符（三套格式）

模板中可能同时存在以下格式的占位符，**全部需要替换**：

| 格式 | 示例 |
|------|------|
| `_[xxx]_` 双下划线 | `_[招标方名称]_`、`_[招标编号]_` |
| `[xxx]` 纯方括号 | `[招标方名称]`、`[项目名称]` |
| `（xxx）` 中文全角括号 | `（标段1名称）` |

**操作**：解包模板扫描所有占位符：

```bash
# PowerShell 解包并扫描
Add-Type -AssemblyName System.IO.Compression.FileSystem
$tmp = "$env:TEMP\scan_tmpl"
[System.IO.Compression.ZipFile]::ExtractToDirectory("模板.docx", $tmp)
Get-ChildItem -Recurse $tmp -Filter *.xml | ForEach-Object {
  $txt = Get-Content $_.FullName -Raw
  [regex]::Matches($txt, `'_[^_]+_`') | % { $_.Value }
  [regex]::Matches($txt, `'\[[^\]]+\]`') | % { $_.Value }
  [regex]::Matches($txt, `'（[^）]+）`') | % { $_.Value }
} | Sort-Object -Unique
```

### 2.2 脱敏文本（公司特定字符串）

脱敏模板中的敏感信息已被替换为虚拟文本。常见模式：
- 公司全称 → `XX重工股份有限公司`
- 公司简称 → `XX重工`
- 法人姓名 → `金红萍`
- 授权代表 → `沈小芳`
- 地址 → `[公司地址]`、`江苏省 苏州市`
- 银行信息 → `[开户银行名称]`、`[银行账号]`
- 联系方式 → `[电话已脱敏]`、`[手机已脱敏]` 等
- 金额占位 → `【 随机金额万美元 ... 】`

**操作**：阅读 `word/document.xml` 和 `word/header*.xml`，找出上述模式的文本。

### 2.3 日期处理（自动，无需手动配键）

**日期由引擎自动处理，不需要在 JSON 中配置。**

> **处理顺序**：阶段1先替换日期 → 阶段2再替换占位符。日期优先处理避免占位符值干扰日期匹配。

引擎通过三套正则覆盖所有日期变体，自动替换为当天实时日期：

| 正则 | 覆盖格式 | 示例 |
|------|---------|------|
| 完整日期（含下划线/空格分隔） | `2014年7月29日`、`__2014__年__7__月__29__日` |
| 有年份无月日数字 | `2014年    月    日`、`__2014__年_____月_____日` |
| 无年份的月日占位 | `年    月    日` |

日期**始终替换为当天**（`YYYY年MM月DD日`），当日日期自动跳过。如招标文件有特定截止日期要求，在 pipeline 解析时会保留。

> ⚠️ **不要**在 JSON 中添加日期碎片键（如 `"2014"`、`"7月"`、`"29日"` 等），这会导致与正则替换冲突产生乱码。

### 2.4 构建占位符清单

汇总为表格，标注每项的**数据来源**：

| 原文（模板中） | 类型 | 需填入 | 来源 |
|---------------|------|--------|------|
| `_[招标方名称]_` | 占位符 | 招标方全称 | 步骤1 |
| `[招标编号]` | 占位符 | 招标编号 | 步骤1 |
| `XX重工股份有限公司` | 脱敏文本 | 投标人全称 | 步骤3 |
| `金红萍` | 脱敏文本 | 法人姓名 | 步骤3 |
| `【 30 】` | 脱敏文本 | 投标有效期 | 步骤1 |
| `【 随机金额... 】` | 脱敏文本 | 实际报价金额 | 步骤3 |
| 日期 `2014年7月29日` | 自动替换 | 今天日期 | 引擎 |

## 步骤5: 执行填充

### > **大文件处理**：模板 >5MB 时自动使用 fill_large 纯 Python zipfile 引擎，零外部依赖，同样支持日期优先 + 占位符后处理。

方式 A：Python（推荐）

```bash
python fill_py.py <模板.docx> <输出.docx> <替换数据.json>
```

两阶段引擎：先占位符替换，再整日期替换。自动报告遗漏占位符。

> ⚠️ **格式注意**：当占位符跨多个 Word run 时，引擎会合并 run 进行替换，差异化格式可能丢失。建议模板占位符使用统一格式。
>
> **页眉页脚处理**：`python-docx` 只能访问默认页眉。引擎保存后会额外解包 DOCX，直接修补 `header*.xml` 和 `footer*.xml` 中遗漏的占位符和日期。

### 方式 B：Node.js（零依赖，仅 Windows）

```bash
node fill_template.js <模板.docx> <输出.docx> <替换数据.json>
```

同两阶段策略，额外自动检查遗漏占位符。依赖 PowerShell 进行 ZIP 操作，不支持 macOS/Linux。

### 方式 C：管道一键生成

```bash
python bid_pipeline.py 招标文件.md [--company-json 公司数据.json] [--templates 模板1.docx 模板2.docx ...]
```

自动完成步骤1→4→5，从招标文件 Markdown 到成品标书。

成品文件默认输出到招标文件同目录下的 成品标书/ 子文件夹。文件命名格式： 标书类型_项目名称.docx。

- 默认查找模板目录下 `*脱敏*.docx` 文件
- 可通过 `--templates` 显式指定模板列表
- `--company-json` 指定公司脱敏数据（格式同步骤4中的公司部分）
- 大文件（>5MB）自动切换纯 Python zipfile 引擎，跨平台兼容

## 步骤6: 验证输出

填充完成后自动运行 verify_bid.py 进行二次核对：

```bash
# 自动（pipeline 内置）
python bid_pipeline.py 招标文件.pdf --company-docx 公司信息.docx
# → 填充完成后自动审核每个输出文件

# 手动
python verify_bid.py 成品标书.docx --company-docx 公司信息.docx

# 跳过审核
python bid_pipeline.py 招标文件.md --company-json company.json --no-verify
```

**检查等级：**

| 等级 | 检查项 | 示例 |
|------|--------|------|
| P0 致命 | 遗漏占位符、日期未更新、公司名不一致 | _[招标编号]_ 未替换 |
| P1 严重 | 关键字段为空、疑似乱码、金额不一致 | 投标报价字段空白 |
| P2 提示 | 空白方括号、签名区提醒、异常标点 | 【  】 未填充 |

手动补充检查：
- 投标函抬头、授权书姓名、保证金账户
- 报价总表金额、签名区
- 页眉页脚信息

## 故障排除

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 占位符未被替换 | JSON 中的 key 与模板中的文本不完全一致 | 检查空格、全角/半角字符 |
| 日期没有被替换 | 日期格式不匹配正则 | 确保日期为"YYYY年MM月DD日"格式 |
| `fill_py.py` 报 ModuleNotFoundError | 未安装 python-docx | `pip install python-docx` |
| `fill_template.js` 报错 | 非 Windows 系统 | 改用 Python 方式 |
| 输出文件格式错乱 | 模板中的复杂格式 | 参见步骤5的格式警告 |
| 保证金表格金额为空 | 模板金额为空白占位符（`￥  元`） | 在 JSON 中精确匹配该空白模式 |
| 页眉未更新 | 模板使用了非默认页眉类型 | 引擎已自动处理，无需手动干预 |
| 模板未找到 | 命名不符合约定 | 使用 `--templates` 显式指定 |

## 严禁事项

- **绝不编造**业绩、资质、人员、报价等任何信息
- **绝不跳过**招标文件要求的任何附件
- **报价金额**必须经用户确认
- 模板中已有的标准条款结构**不要修改**，只替换变量
- **不要在 JSON 中配置日期碎片键**，日期由引擎自动整日期替换
