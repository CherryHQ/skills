# 评分量表

## 单题四档

| 档位 | 含义 | 判定线 |
|---|---|---|
| **优** | 答到点 + 有结构 + 有亮点 | 命中考点，有具体案例/数据，结构清晰，甚至超出预期主动延伸 |
| **良** | 答对了但不出彩 | 命中考点，但案例薄、结构平、没把自己的优势打出来 |
| **中** | 沾边但没说透 | 方向对但答得浅 / 散 / 跑偏一半，或被追问后才补上 |
| **差** | 没答到 / 答崩 / 答不上 | 偏题、卡壳、被追问接不住、或硬伤（讲错、与简历矛盾） |

## 5 维画像量表（每维 1–5 分）

| 分 | 状态 |
|---|---|
| 5 | 明显优势，面试官大概率加分 |
| 4 | 稳，达标偏上 |
| 3 | 及格，不扣分也不加分 |
| 2 | 短板，可能成为拒绝理由之一 |
| 1 | 硬伤，大概率致命 |

5 个维度：
1. 专业/技术深度
2. 表达结构（STAR）
3. 岗位匹配（对准 JD + 打出卖点）
4. 临场反应（追问 / 压力题 / 没准备的题）
5. 反问质量（最后提问环节）

## STAR 结构检查（用于「表达结构」维度）

- **S**ituation 背景：在什么场景下
- **T**ask 任务：你要解决什么 / 你的职责
- **A**ction 行动：你具体做了什么（重点，要体现个人贡献而非「我们」）
- **R**esult 结果：量化结果 / 影响

回答缺哪一环就在诊断里点名，尤其是 A（只说团队不说自己）和 R（没有结果）是最常见的丢分点。
