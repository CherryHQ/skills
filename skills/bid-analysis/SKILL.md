# 标书分析（Bid Analysis v3.0）

## 技能说明

对招标文件（PDF/DOCX/DOC/XLSX）进行结构化解析，提取关键条款、计算评分、检查合规性、识别废标风险，最终生成 Markdown 分析报告并自动保存至源文件所在目录。

**触发条件：**
- 用户上传 `.pdf` `.docx` `.doc` `.xlsx` 格式的招标文件
- 用户说"分析标书" "评估投标文件" "检查标书合规性"
- 用户询问"标书评分" "投标条款" "技术方案评估"
- 用户需要对比多份标书或提取标书中的关键信息

**执行规则：** 触发后，必须完成以下三步流程，并在最后将分析报告保存为 `.md` 文件。

---

## 工作流程

```
读取文档 → 提取信息 → 深度分析 → 生成报告.md
```

---

## 第一步：读取文档

根据文件格式选择读取策略：

| 格式 | 策略 |
|------|------|
| PDF（文字型） | 直接提取文本，按页码分段，保留原始段落结构 |
| PDF（扫描件） | 逐一描述每页可见内容。扫描件置信度不足时，标注「OCR置信度：低」，保留原始截图描述 |
| DOCX/DOC | 提取正文、表格、页眉页脚。表格保留行列结构 |
| XLSX | 读取所有工作表，保留表头和数据行 |

**读取完成后输出：** 文档总页数、章节结构（目录或标题层级）、关键表格数量。

---

## 第二步：信息提取

从文档中逐项提取以下内容，填入结构化表格：

### 2.1 基本信息

| 提取项 | 查找位置 | 输出格式 |
|--------|----------|----------|
| 项目名称 | 封面、第一章 | 原文完整名称 |
| 招标编号 | 封面、投标须知 | 原文编号 |
| 招标方 | 封面、投标须知前附表 | 单位全称 |
| 招标代理 | 封面、投标须知 | 单位全称 |
| 采购预算/控制价 | 投标须知、采购需求 | 金额（万元） |
| 资金来源 | 投标须知 | 财政/自筹/混合 |
| 招标方式 | 投标须知 | 公开招标/邀请招标/竞争性磋商等 |

### 2.2 时间节点

| 提取项 | 输出格式 |
|--------|----------|
| 公告发布日期 | YYYY年MM月DD日 |
| 投标截止时间 | YYYY年MM月DD日 HH:MM |
| 开标时间 | YYYY年MM月DD日 HH:MM |
| 投标有效期 | X 日历日（从开标日起算） |
| 答疑截止时间 | YYYY年MM月DD日（如有） |
| 保证金缴纳截止 | YYYY年MM月DD日（如有） |

### 2.3 资质要求

提取所有资质条款，区分硬性（必须满足）和加分项：

| 资质类型 | 要求原文 | 性质 | 是否一票否决 |
|----------|----------|------|-------------|
| 营业执照 | ... | 硬性 | 是 |
| 行业资质（如ISO、CMMI） | ... | 硬性/加分 | 视条款 |
| 业绩要求 | ... | 硬性/加分 | 视条款 |
| 人员证书 | ... | 硬性/加分 | 否 |
| 财务要求 | ... | 硬性 | 视条款 |

**每个资质标注：** ①能否用复印件 ②是否需要原件核查 ③是否接受联合体成员提供。

### 2.4 评分标准

逐条提取评分规则，格式为：

```
评分项 | 满分 | 评分细则 | 我方预估得分 | 预估依据
```

对每条细则做三件事：
1. 判断是客观分（有就是满分、没有就是零）还是主观分（评委打分有浮动）
2. 客观分标注「X分/确定性高」；主观分标注「X分/浮动±Y分」
3. 列出满分需要提供的证明材料清单

### 2.5 商务条款

| 条款类型 | 招标要求 | 偏离是否允许 | 偏离后果 |
|----------|----------|-------------|----------|
| 付款方式 | ... | 是/否 | ... |
| 质保期 | ... | 是/否 | ... |
| 交货期/工期 | ... | 是/否 | ... |
| 联合体 | 接受/不接受 | — | ... |
| 分包 | 允许/不允许 | — | ... |

### 2.6 技术规格

提取技术参数要求，对照我方能力标注：

| 技术参数 | 招标要求 | 我方是否满足 | 证明材料 |
|----------|----------|-------------|----------|
| ... | ... | 满足/部分满足/不满足 | ... |

---

## 第三步：深度分析

### 3.1 废标风险清单

逐条审查以下一票否决项：

- [ ] 投标有效期不足
- [ ] 保证金金额不足或缴纳方式不符
- [ ] 签字盖章要求（法定代表人签字+盖章缺一不可的条款）
- [ ] 资质证书过期
- [ ] 联合体协议缺失（如接受联合体）
- [ ] 投标函格式——是否存在任何"不得修改"的固定格式要求
- [ ] 是否存在"不接受负偏离"的条款（技术或商务）
- [ ] 密封要求——是否有与常规不同的装订/密封规定
- [ ] 是否存在招标文件"歧义条款"可能导致废标（如前后矛盾的资质要求）

对每条标注风险等级（致命/高/中），并给出避险操作建议。

### 3.2 技术难点识别

从技术规格中识别：

1. 列出我方不满足或部分满足的技术参数
2. 标注是否可以通过方案弥补（如外包、合作、替代方案）
3. 无法弥补的标注为「硬伤」，提示评估是否继续投标

### 3.3 条款冲突检测

跨章节对比同主题条款：

- 示例：前附表写"质保期3年"、合同草案写"质保期2年"→ 标注冲突，提示必须澄清
- 示例：评分标准要求"3个同类业绩"、投标须知要求"5个"→ 标注冲突

### 3.4 投标策略建议

综合以上分析，给出：

1. **投标可行性判断：** 建议投标 / 谨慎投标 / 不建议投标
2. **得分预估：** 我方预估总分 / 满分
3. **关键得分项：** 列出得分确定性最高的3-5项
4. **关键失分项：** 列出我方薄弱的3-5项及补救建议
5. **报价策略建议：** 基于评分规则（如低价优先 or 均价优先 or 综合评分）给出报价区间建议

---

## 输出：分析报告

将以上所有提取和分析结果，按以下模板生成报告。**报告文件自动保存为 `[项目名称]_分析报告.md`，存放在招标文件所在目录。**

```
# [项目名称] 投标分析报告

> 生成时间：[当前日期时间] | 招标文件：[文件名]

## 一、基本信息摘要
[填入 2.1 的内容，表格形式]

## 二、时间节点梳理
[填入 2.2 的内容，表格形式]
> ⚠️ 关键提醒：距离投标截止还有 [计算天数] 天

## 三、资质要求清单
[填入 2.3 的内容，表格形式]
> 我方满足：X/X 项 | 不满足：0 项 | 待确认：Y 项

## 四、评分标准深度解析
[填入 2.4 的内容，逐条]
> 我方预估总分：XX / XXX | 客观分确定性：XX%

## 五、商务条款与投标要求
[填入 2.5 的内容，表格形式]
> 是否存在负偏离：是/否

## 六、技术方案对照
[填入 2.6 的内容，表格形式]
> 技术满足率：X/X 项 | 硬伤：X 项

## 七、废标风险清单
[填入 3.1 的内容，逐条标注风险等级]
> 致命风险：X 项 | 高风险：X 项 | 中风险：X 项

## 八、技术难点与补救建议
[填入 3.2 的内容]

## 九、条款冲突提示
[填入 3.3 的内容。如无冲突则写"未发现条款冲突"]

## 十、投标策略建议
[填入 3.4 的内容]
> **最终建议：[建议投标 / 谨慎投标 / 不建议投标]**

## 附注：未识别条款
[列出未被归入以上任何类别的条款原文及其所在位置]
> 注：以下条款无法被自动归类，需人工复核。
```

## 分析模式

根据用户需求或文档复杂度，选择分析深度：

**快速扫描（5分钟）：**
- 提取 2.1 基本信息 + 2.2 时间节点 + 3.1 废标风险
- 适用于：快速判断是否值得投标

**标准分析（15分钟，默认）：**
- 全部 2.1-2.6 提取 + 3.1 废标风险 + 3.3 条款冲突
- 适用于：常规投标决策

**深度分析（30分钟）：**
- 全部提取 + 全部深度分析 + 3.4 策略建议
- 适用于：重大项目、竞争激烈的标的

---

## 异常处理规则

| 情况 | 处理 |
|------|------|
| 条款无法归类 | 记录原文片段及所在位置（页码/段落），列入「附注：未识别条款」，**不得默默跳过** |
| 条款前后矛盾 | 提取冲突双方的原文对照，列入「九、条款冲突提示」 |
| 扫描件文字模糊 | 标注置信度等级（高/中/低），置信度低的标注"需人工核对原始文件" |
| 条款引用外部附件 | 记录引用位置和被引用文件信息，提示用户手动检查附件内容 |
| 表格/图片内嵌文字 | 尝试描述可见内容，失败则记录位置并标注"包含无法解析的表格/图片" |
| 源文件损坏或无法读取 | 直接告知用户原因，不生吞活剥 |
| 源文件为空 | 提示用户文件无效 |
| 报告内容过长 | 保持 10 章节核心结构，精简要但不删除任何未识别条款的警告 |

**核心原则：所有无法识别或存疑的条款，必须以「附注」形式明确记录。用户应当清楚知道哪些内容被完整分析、哪些内容需人工复核。**

---

## 硬性约束

- **绝不编造数据。** 缺失的信息写"招标文件未明确"，不得推测
- **不修改招标文件原文。** 引用条款时使用原文，不概括、不转译
- **签字盖章要求必须逐字核对。** "法定代表人签字并加盖公章"和"法定代表人或授权代表签字"是截然不同的要求
- **时间计算使用实际日期。** 用当前系统时间推算截止天数
- **评分预估标明不确定性。** 主观分标注浮动范围，不说死
- **报告保存后，在对话中告知用户完整文件路径**
- **若存在未识别条款或检测到条款冲突，额外提醒用户关注「附注」和「条款冲突提示」章节**
